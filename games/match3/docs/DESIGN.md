# Match-3 游戏设计文档

## 一、游戏总览

一款以**算法驱动**为核心的三消游戏原型。不同于传统三消手游依赖大量手工关卡设计，本项目所有关卡参数（难度、目标、规则变体）均由算法生成，配合动态难度调整（DDA）系统，实现「每个玩家都有恰好合适的挑战」。

**核心产出：**

| 系统 | 解决的问题 | 对标竞品 |
|------|-----------|---------|
| Smart Drop 智能掉落 | 根据玩家水平调控运气 | Candy Crush 持有类似专利 |
| 渐近难度曲线 | 避免线性增长带来的前期无聊/后期断崖 | Royal Match 手工调参 |
| 关卡修饰器 | Lv40+ 横向多样性，打破重复感 | Rogue-like 元素融合 |
| 多目标 + 星级 | 深化单关可玩性，增加重玩动力 | Candy Crush 星级系统 |
| AI 自动验证 | 3 种 AI 策略 benchmark，数据驱动平衡调优 | 内部工具 |

## 二、基础玩法

### 2.1 棋盘与消除

- 默认 **8×8** 棋盘（部分关卡 6×6 或 9×9）
- **7 种宝石**，每种有独特形状和颜色，无需纹理贴图即可辨识
- 玩家点击选中一颗宝石，再点击相邻宝石完成交换
- 交换后如果形成 **3 颗及以上同色连线**（水平或垂直），触发消除
- 消除后上方宝石受重力下落，顶部生成新宝石
- 下落后自动检测新的匹配 → **连锁反应（Cascade）**
- 若交换不产生任何匹配，自动还原

### 2.2 特殊宝石

参照 Candy Crush 经典设计，匹配更多宝石产生特殊宝石：

| 触发条件 | 产生物 | 激活效果 |
|----------|--------|---------|
| **4 连线** | 条纹宝石 | 消除整行 或 整列 |
| **L/T 形交叉** | 包装宝石 | 3×3 范围爆炸 |
| **5 连线** | 彩色炸弹 | 消除棋盘上所有指定颜色 |

特殊宝石之间还能组合触发更强效果：

| 组合 | 效果 |
|------|------|
| 条纹 + 条纹 | 十字消除（一行 + 一列） |
| 条纹 + 包装 | 大十字（3行 + 3列） |
| 包装 + 包装 | 5×5 大范围爆炸 |
| 彩色炸弹 + 任意 | 消除该颜色全部宝石 |

### 2.3 计分规则

```
单次得分 = 消除数量 × 10 × combo倍率 × scoreMultiplier
```

- 首次匹配 combo=1，每次连锁 +1（×2, ×3...）
- 连锁结束后 combo 重置
- `scoreMultiplier` 默认 1.0，由关卡修饰器调整（如 fragile 为 1.5）

### 2.4 通关与失败

- **通关**：在步数耗尽前达成主目标分数
- **失败**：步数用完未达标，或棋盘无有效移动

## 三、难度曲线设计

### 3.1 设计哲学

传统三消的线性难度增长有两个问题：
1. **前期太无聊** — 参数增长慢，缺乏挑战
2. **后期断崖** — 参数线性增长，但玩家技能增长是对数型的

我们采用**分阶段渐近曲线**：每个阶段有独立的天花板，目标分数快速增长后趋平，在阶段切换时**主动降低目标**给予「喘息期」。

### 3.2 三个阶段

```
        目标分数
  2800 ┤                                          ···········  ← 7色天花板
       │                                    ╱
  2050 ┤                  ·············╱····   ← 6色天花板
       │              ╱
  1000 ┤ ···········╱  ← 5色天花板
       │        ╱
   300 ┤───╱
       └──┬──┬──────┬──────────────┬────────────
         Lv1 Lv8   Lv9           Lv24          Lv25+
              ↑ 目标回落           ↑ 目标回落
         5种宝石    6种宝石            7种宝石
```

| 阶段 | 关卡 | 宝石数 | 目标范围 | 步数范围 | 定位 |
|------|------|--------|---------|---------|------|
| 学习期 | Lv1-8 | 5 | 300→1000 | 32→26 | 熟悉操作，建立信心 |
| 核心期 | Lv9-24 | 6 | 950→2050 | 25→20 | 掌握特殊宝石策略 |
| 专家期 | Lv25+ | 7 | 1700→2800 | 22→18 | 高阶策略 + 修饰器变体 |

**渐近公式**（以 6 色阶段为例）：

```
target = 850 + 1200 × (1 - e^(-0.1 × phase))
```

其中 `phase = levelNum - 8`。这使得：
- 前几关增长快（每关 +80~100 分）
- 后期趋平（每关仅 +10~20 分）
- 永远不会超过天花板 2050

### 3.3 步数衰减

步数同样使用指数衰减，向阶段地板趋近：

```
moves = floor + (32 - floor) × e^(-0.06 × (levelNum - 1))
```

| 阶段 | 地板 | 效果 |
|------|------|------|
| 5 色 | 26 步 | 宽裕，专注学习 |
| 6 色 | 20 步 | 需要效率意识 |
| 7 色 | 18 步 | 每步都关键 |

**阶段过渡奖励**：切换宝石数时额外 +3/+2/+1 步，平滑难度跳跃。

## 四、Smart Drop 与 DDA 系统

### 4.1 Smart Drop — 智能掉落算法

这是本项目的**核心技术亮点**。当宝石消除后新宝石从顶部掉落时，不再使用纯随机，而是根据棋盘状态和 bias 参数智能选择颜色。

**算法流程：**

```
对于每个空位 (row, col)：
  1. 为每种颜色计算基础权重 w = 1.0
  2. 分析该位置放置该颜色后的匹配潜力：
     - 0：不形成任何连续  → 权重不变
     - 1：形成 2 连（near-match）→ 根据 bias 调整
     - 2：形成 3+ 连（直接匹配）→ 根据 bias 调整
  3. bias > 0（助力模式）:
     - near-match: w += bias × 2.0  （更有用，给更多权重）
     - 直接匹配: w += bias × 1.0
  4. bias < 0（挑战模式）:
     - 直接匹配: w *= max(0.1, 1.0 + bias)  （大幅降低但不归零）
     - near-match: w *= max(0.3, 1.0 + bias × 0.5)
  5. 加权随机选择
```

**设计要点：**
- 权重永远 > 0 — 任何颜色都有可能出现，避免玩家觉得「被操控」
- bias=0 时完全退化为均匀随机
- 助力模式更偏好 near-match（2 连），因为它为玩家「创造机会」而非直接送分
- 挑战模式减少但不消除匹配机会，保持可玩性

### 4.2 DDA 动态难度调整

在 Smart Drop 的 bias 基础上，叠加一层根据**玩家表现**的自适应调整。

**触发机制：** 同一关连续失败次数（`failCount`）驱动 bias 偏移。

**三条可选曲线：**

```
    bias 偏移
 0.6 ┤            ──────── linear（稳定增长）
     │          ╱·········· logarithmic（前快后慢）
 0.4 ┤        ╱·╱
     │      ╱·╱          quadratic（前慢后快）
 0.2 ┤    ╱·╱         ╱
     │  ╱·╱       ╱
   0 ┤╱·╱─────╱──────────
     └─┬──┬──┬──┬──┬──┬──
       0  2  4  6  8  10  failCount
```

| 曲线 | 公式 | 特点 |
|------|------|------|
| linear | `0.05 × n` | 每次失败固定 +0.05 |
| quadratic | `0.015 × n²` | 前几次几乎无感，后期急剧增加 |
| logarithmic | `0.12 × ln(n+1)` | 首次失败即有明显帮助，后期趋平 |

**最终 bias = 关卡基础 bias + DDA 偏移**，限制在 [-0.5, 0.6]。

**关卡基础 bias：**

| 关卡范围 | 基础 bias | 意义 |
|----------|----------|------|
| Lv1-3 | +0.25 | 新手教学，积极助力 |
| Lv4-8 | +0.10 | 学习期，轻微助力 |
| Lv9-16 | 0.00 | 6 色适应期，纯随机 |
| Lv17-24 | -0.05 | 轻微挑战 |
| Lv25-30 | +0.05 | 7 色补偿 |
| Lv31+ | 0.00 | 专家中立 |

**效果示例：** 一个 Lv20 的玩家连续失败 5 次：
- 基础 bias = -0.05
- DDA (linear) = 0.05 × 5 = +0.25
- 最终 bias = +0.20 → 棋盘会「悄悄」多出一些匹配机会

### 4.3 为什么不直接降低目标分数？

直接降低目标会让玩家意识到「游戏在放水」。Smart Drop 的优势在于**不可感知性**——玩家只会觉得「这局运气好」，而非「游戏变简单了」。这正是 Candy Crush 等头部产品使用类似技术的原因。

## 五、Resolve 算法 — 匹配检测与连锁处理

### 5.1 匹配检测 (`Grid.findMatches`)

每次交换或下落后，扫描整个棋盘：

```
1. 水平扫描：逐行查找 3+ 连续同色宝石，记录为 hRuns
2. 垂直扫描：逐列查找 3+ 连续同色宝石，记录为 vRuns
3. 交叉检测：hRun 和 vRun 在同一位置交叉 → L/T 形 → 包装宝石
4. 剩余 run 处理：
   - 5+ 连 → 彩色炸弹（放在玩家交换位置或连线中央）
   - 4 连 → 条纹宝石（方向垂直于匹配方向）
   - 3 连 → 无特殊
```

### 5.2 特殊宝石激活 (`Grid.activateSpecials`)

匹配集合中如果包含特殊宝石，触发链式反应：

```
repeat:
  遍历 matched 集合中所有特殊宝石
  对每个未处理的特殊宝石：
    striped_h → 将整行所有宝石加入 matched
    striped_v → 将整列所有宝石加入 matched
    wrapped   → 将 3×3 范围所有宝石加入 matched
  如果本轮有新宝石被加入 → 继续循环（新加入的可能也是特殊宝石）
until 无新增
```

这实现了**递归连锁**：条纹宝石清行时碰到另一个特殊宝石，后者也会被激活。

### 5.3 完整 Resolve 循环

```
玩家交换两颗宝石
  ↓
检测特殊交互（彩色炸弹/特殊+特殊组合）
  ↓
findMatches → activateSpecials → removeMatches（缩放+粒子动画）
  ↓
clearRemoved → spawnSpecials → applyGravity（下落+新宝石生成）
  ↓
再次 findMatches...（连锁检测）
  ↓
循环直到无匹配 → 判定胜负 → idle
```

每个步骤之间通过 **Tween 动画系统**分隔，确保玩家能看清每一步发生了什么。

## 六、Lv40+ 长线留存系统

### 6.1 问题诊断

Benchmark 数据显示 Lv40 后难度趋近天花板（目标渐近收敛 + 步数触底），玩家面临「相同体验反复」的问题。纵向难度已经没有空间，需要**横向多样性**。

### 6.2 关卡修饰器（Level Modifiers）

每个关卡随机获得 0-2 个修饰器，从 Lv10 开始出现。**相同关卡号永远产生相同修饰器**（确定性种子 RNG），保证公平性和可重玩性。

| 修饰器 | 效果 | 设计意图 |
|--------|------|---------|
| `cascade_king` | 连锁得分额外 +50% | 奖励深度思考 |
| `color_limit` | 宝石种类 -1，目标 +20% | 更易匹配但要求更高分 |
| `big_board` | 棋盘 9×9 | 更多空间，更长的连锁 |
| `small_board` | 棋盘 6×6，目标 -25% | 紧凑空间，精准操作 |
| `no_specials` | 不生成特殊宝石 | 纯基础功考验 |
| `special_start` | 开局 3 个特殊宝石 | 鼓励激进策略 |
| `fragile` | 步数 -25%，基础分 ×1.5 | 高风险高回报 |
| `generous` | 掉落助力 +0.2，目标 +15% | 运气好但要求高 |

**分配规则：**

| 关卡段 | 修饰器数 | 可选池 |
|--------|---------|-------|
| Lv1-9 | 0 | — |
| Lv10-24 | 1 | 温和池（cascade_king, color_limit, special_start, generous） |
| Lv25-49 | 1-2 | 全池 |
| Lv50+ | 2 | 全池 |

**互斥规则**保证不出现不合理组合（如 big_board + small_board、fragile + small_board）。

### 6.3 多目标系统

从 Lv15 开始，约 60% 的关卡附带一个次要目标：

| 目标类型 | 示例 | 追踪方式 |
|----------|------|---------|
| `score` | 达到 1800 分（主目标，必选） | States.score |
| `collect` | 消除 15 颗蓝色宝石 | States.collected[type] 计数 |
| `combo` | 达成 4 连击 | States.maxCombo 追踪 |
| `specials` | 创建 5 个特殊宝石 | States.specialsCreated 计数 |
| `moves_left` | 剩余 5 步以上通关 | States.movesLeft |

### 6.4 星级评价

```
★★★  完成所有目标（主目标 + 次要目标）
★★☆  完成主目标 + 效率过关（使用 ≤75% 步数）
★☆☆  仅完成主目标
```

星级存储在存档中（`Save.data.levelStars[levelNum]`），保留历史最佳。玩家可以重玩已过的关卡刷星。

## 七、AI 验证系统

### 7.1 三种 AI 策略

| 策略 | 算法 | 强度 | 用途 |
|------|------|------|------|
| **greedy** | 评估每种交换的即时得分（匹配数+特殊宝石 bonus） | 弱 | 模拟新手玩家 |
| **heuristic** | greedy + 单次连锁模拟 + 棋盘潜力评估 | 中 | 模拟普通玩家 |
| **montecarlo** | 每个候选 move 做 15 次随机连锁模拟取平均 | 强 | 模拟高手玩家 |

### 7.2 Benchmark 验证

运行命令：`"D:/Program Files/LOVE/lovec.exe" F:/love2d --benchmark`

自动跑 3 策略 × 3 DDA 曲线 = 9 种组合，每种最多到 100 关，输出 CSV 数据。

**最新 Benchmark 结果：**

| 策略 | 平均最高关 | 定位 |
|------|-----------|------|
| greedy | Lv29 | 新手天花板 |
| montecarlo | Lv35 | 中等玩家天花板 |
| heuristic | Lv37 | 高手天花板 |

**分段通过率（全策略汇总）：**

| 关卡段 | 首次通过率 | 最终通过率 |
|--------|-----------|-----------|
| Lv1-9 | 100% | 100% |
| Lv10-24 | 84% | 100% |
| Lv25-34 | 37% | 94% |
| Lv35+ | 6% | 75% |

**解读：**
- 前 24 关：所有水平的 AI 都能通过，说明学习期和核心期难度适当
- Lv25-34：首次通过率低但最终都能过，说明 DDA 系统有效运作
- Lv35+：开始有 AI 卡关，这正是需要修饰器和目标系统提供多样性的区间

## 八、技术亮点总结

| 技术 | 实现方式 | 价值 |
|------|---------|------|
| 渐近难度曲线 | 指数衰减公式，阶段天花板 | 永不断崖，无需手动调参 |
| Smart Drop | 棋盘状态分析 + 加权随机 | 不可感知的难度调控 |
| DDA 三曲线 | linear/quadratic/logarithmic | 适配不同玩家群体 |
| 修饰器系统 | 确定性种子 RNG + 互斥规则 | 公平的横向多样性 |
| AI Benchmark | 3 策略 × 3 曲线自动验证 | 数据驱动，消除拍脑袋调参 |
| 全动画 Resolve | Tween 驱动每步可视化 | 玩家始终清楚发生了什么 |

## 九、体验指南

### 启动游戏

```
"D:/Program Files/LOVE/love.exe" F:/love2d
```

### 操作说明

1. 输入昵称（3-12 位字母数字）→ 回车开始
2. 点击选中宝石（白色方框高亮）→ 点击相邻宝石交换
3. 注意观察：顶部 HUD 显示目标分数和剩余步数
4. 有次要目标时，进度条下方会显示目标要求和完成状态

### 调试面板（F1 键）

按 **F1** 打开 GM 面板，可以：
- 查看当前 DDA 曲线、bias 值、失败次数
- 查看当前关卡的修饰器和目标进度
- 切换 DDA 曲线（linear/quadratic/logarithmic）
- 重置失败计数
- 开启 Auto-Play（AI 自动打关）并选择策略

### 建议体验路线

1. **Lv1-5**：体验基础三消，观察连锁反应和得分
2. **Lv9**：宝石种类从 5 → 6，注意目标回落
3. **Lv10-15**：开始出现修饰器（F1 面板查看）
4. **Lv15+**：出现次要目标（如「消除 15 颗蓝色宝石」）
5. **故意失败 3-5 次**：观察 DDA 系统介入（F1 面板中 bias 值上升）
6. **开启 Auto-Play**：用 montecarlo 策略观看 AI 表现

### 运行测试

```
"D:/Program Files/LOVE/lovec.exe" F:/love2d --test
```

96 项测试，覆盖匹配检测、特殊宝石、关卡生成、修饰器分配、目标系统、存档、AI 策略等。
